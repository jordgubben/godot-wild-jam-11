Godot Open Dialogue

About

The Godot Open Dialogue is a non-linear conversation system created by J. Sena and released as CC-BY. This system was created using the 3.1 alpha 5 version of the Godot Engine. You can change the code to make it work on previous versions but things may not work as they should.

The block types

In order to define the various possible behaviors during a conversation the system has a few sets of parameters called blocks. Each block has its own ways to be customized and connected to other blocks in order to create a rich dialogue experience. You can see below the block types and the way they can be used.

1. text

The most common type of dialogue block probably will be the text one. This is the simplest way to print a string in the players screen.

The parameters

type Must be set to "text".
content Here goes the string to be parsed and printed.
next You can set the name of the next dialogue block. If the current block doesn't have a "next" parameter it will be the last dialogue block to be shown.

Code example:

"block_name": { "type": "text", "content": "This string will be parsed and printed", "next": "next_block_name" }

2. divert

This type can be used when you want to show certain blocks based on conditions. Right now it only supports the "variable" condition but it will be exapnded in the future.

The parameters

type Must be set to "divert".
condition Only supports "variable" for now.
variable Name of the variable to be checked for true or false.
true The name of the block to be shown if the variable returns true.
false The name of the block to be shown if the variable returns false.

Code example:

"block_name": { "type": "divert", "condition": "variable", "variable": "variable_name", "true": "block_name_if_true" "false": "block_name_if_false" }

3. question

This type allows you to have a question with multiple options for the player to choose. Based on the answer you can call another block of any type.

The parameters

type Must be set to "question".
text Here goes the question string to be parsed and printed as a question. If the typewriter effect is active the player will only be allowed to select an option once the script has done printing the entire text.
options Here goes an array of strings to be displayed as choices. You can use as many options as you like.
next Here goes an array of strings with the names of the blocks to show based on the player's answer. Important: This array must have the same size of "options".

Code example:

"block_name": { "type": "question", "text": "String to be parsed and printed.", "options": ["Text for option 1", "Text for option 2"], "next": ["block_for_option_1", "block_for_option_2"] }

4. action

This type allows you to perform an action without having to print anything. For now it just updates variables.

The parameters

type Must be set to "action".
text Here goes a string to be parsed and printed. If the block has no "text" parameter it will execute the action and then, if available, go to the "next" block.
value Here goes three values. The first one must be a string with the name of the action. Right now the only action available is "variable". The second one is the key (or keys) that is going to be updated. It can be a string or an array of strings. The third one is the value (or values) to be assigned to the key(s) defined before. It can be a string or an array of strings. Important: if you use a string in the second value and an array in the third one, the variable's value will be the array itself. If you use an array on the second value, you must use an array with the same size as the third value.
next You can set the name of the next dialogue block. If the current block doesn't have a "next" parameter it will be the last dialogue block to be shown.

Code example:

"block_name": { "type": "action", "text": "String to be parsed and printed.", "value": ["variable", "name_of_the_variable", "value"], "next": ["next_block_name"] }
